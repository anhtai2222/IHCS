# Agent Rules

- Use the repository as the source of truth
- Read `.ai/` context files before making changes
- Prefer minimal, correct, maintainable solutions
- Avoid unrelated refactors
- Do not introduce major dependencies without clear reason
- Keep changes consistent with the existing architecture
- Preserve working behavior unless the task explicitly requires changes
- When editing code, prefer incremental updates over large rewrites
- If uncertain, choose the safest small change and state assumptions clearly
